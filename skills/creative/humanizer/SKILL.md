# 휴머나이저 (The Humanizer)

AI 초안을 빠르게 쓰되, 내가 쓴 것처럼 들리게 만든다. 두 가지 일을 한다:
1. **감사(Audit)**: 텍스트에서 AI 글쓰기 특징을 찾아 표시한다.
2. **재작성(Rewrite)**: 그 특징들을 제거하고 자연스러운 산문으로 다시 쓴다.

상세 특징 목록:
- 영어: `references/ai-tells.md` (30+ 패턴)
- 한국어: `references/ko-tells.md`

## 입력 처리

사용자가 텍스트를 주면 (붙여넣기/파일/메시지) 바로 작업한다. 모드:
- 기본: **감사 + 재작성** 둘 다.
- "감사만" / "audit only" → 표시만, 재작성 생략.
- "재작성만" / "just rewrite" → 깔끔한 결과만, 설명 생략.

언어는 입력 텍스트를 따른다. 한국어 입력이면 한국어로, 영어면 영어로.

## 워크플로우

### 1단계 — 감사

`references/`의 특징 목록과 대조해 발견된 것을 나열한다. 위치(인용)와 유형을 함께.

```
발견된 AI 특징 (7건):
1. em-dash 남용 — "그것은 단순한 도구가 아니라 — 혁명입니다" (1곳)
2. tricolon 강박 — "빠르고, 강력하고, 직관적인" (2곳)
3. "~할 수 있습니다" 반복 (4곳)
4. 영어 번역체 수동태 — "최적화될 수 있도록 설계되었습니다"
...
```

### 2단계 — 재작성

발견된 특징을 제거하고 다시 쓴다. **규칙**:
- 의미·사실·정보는 100% 보존. 길이는 비슷하게 (요약 아님).
- 문장 길이를 섞는다 (AI는 균일하게 중간 길이를 쓴다). 짧은 문장으로 끊어라.
- 완벽한 평행 구조를 깬다. tricolon을 둘 또는 넷으로 바꾸거나 풀어쓴다.
- em-dash → 쉼표/마침표/괄호로 분산. 한 단락에 em-dash 최대 1개.
- 클리셰 동사/형용사(leverage, robust, delve, 원활한, 강력한) → 구체적 단어로.
- 헤징("~라고 할 수 있습니다", "it's worth noting") 삭제. 단언하라.
- 기계적 전환어("또한", "더불어", "Furthermore", "Moreover") 줄이고 흐름으로 연결.
- 결론 상투구("결론적으로", "In conclusion") 삭제.

### 3단계 — 출력

```
## 재작성

[자연스러운 결과]

---
바뀐 핵심: tricolon 2곳 해체, em-dash 3→1, 클리셰 동사 4개 교체.
```

"재작성만" 모드면 결과 텍스트만 출력한다.

## 톤 보존 (CRITICAL)

휴머나이즈는 "AI 티 제거"이지 "내 목소리로 둔갑"이 아니다. 원문의 격식 수준(반말/존댓말/B2B 격식)은 유지한다. 과하게 캐주얼하게 만들지 마라 — 그것도 부자연스럽다.

QJC 맥락에서 자주 쓰는 곳:
- 블로그/뉴스레터 초안 다듬기
- 고객 이메일 (단, `email.md` 톤 규칙 우선)
- SNS 게시물 (스와이프 파일과 연계)

## 본인 스타일 학습 (확장점)

`~/.hermes/automation-state/humanizer-voice.json` 에 사용자가 선호/비선호 표현을 누적하면
재작성이 점점 본인 목소리에 맞춰진다. 사용자가 "이 표현 좋아" / "이건 내 스타일 아니야"
라고 하면 해당 항목을 기록한다. (스와이프 파일의 fingerprint 와 연동 가능.)

## 안티패턴 (재작성이 빠지기 쉬운 함정)

- 재작성 결과에 또 AI 특징을 넣기 (특히 결론 상투구). 자기 출력도 감사하라.
- 정보를 빼서 "자연스럽게" 만들기 — 길이를 줄이지 마라, 표현만 바꿔라.
- 모든 문장을 짧게 — 단조로움도 부자연스럽다. **길이를 섞어라.**
- 한국어인데 영어 특징 목록을 적용 (또는 그 반대). 언어 맞춰 참조.
